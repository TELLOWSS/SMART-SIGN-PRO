# Gemini 기반 보강 백로그

이 문서는 Gemini 분석 결과를 실제 작업으로 전환하기 위한 백로그다.

## 우선순위 규칙
- P0: 안정성/데이터 손상 방지
- P1: 성능/메모리
- P2: UX/유지보수성
- P3: 확장 기능

---

## P0 (반드시 먼저)

### 1) Export 흐름 안전화
- 대상: `App.tsx`, `services/excelService.ts`
- 내용:
  - 내보내기 실패 케이스를 유형별로 분리
  - 사용자 메시지와 디버그 메시지 분리
  - 재시도 시 상태 꼬임 방지
- 완료 기준:
  - 실패 원인이 사용자에게 명확히 노출됨
  - 동일 파일 재시도 3회 이상 시에도 정상 동작

### 2) 병합셀/인쇄영역 보존 회귀 방지
- 대상: `services/excelService.ts`
- 내용:
  - 병합셀 보존 검증 함수 분리
  - 인쇄영역 파싱 단일 유틸화
- 완료 기준:
  - 대표 템플릿 3개에서 병합셀/인쇄영역 불일치 0건

### 3) Object URL 수명 관리 정리
- 대상: `App.tsx`, `services/alternativeExportService.ts`
- 내용:
  - 생성 지점/해제 지점 명확화
  - reset/unmount 시 누수 방지
- 완료 기준:
  - 반복 업로드/리셋 후 브라우저 메모리 증가폭이 완만

---

## P1 (성능)

### 4) 대용량 처리 분기 추가
- 대상: `App.tsx`, `services/excelService.ts`
- 내용:
  - 파일 크기/행 수 기준으로 안전 모드 적용
  - 회전/렌더링 옵션 축소 모드 제공
- 완료 기준:
  - 5MB+ 파일에서 브라우저 멈춤 빈도 감소

### 5) 이미지 회전 캐시 전략 개선
- 대상: `services/excelService.ts`
- 내용:
  - 캐시 키/해제 정책 명확화
  - 동일 회전값 재사용 최적화
- 완료 기준:
  - 다중 배치에서 처리 시간 단축

### 6) 로그 레벨 체계 도입
- 대상: 전체
- 내용:
  - 개발/운영 로그 분리
  - 민감정보 마스킹
- 완료 기준:
  - 운영 모드에서 과도한 로그 제거

---

## P2 (UX/유지보수)

### 7) App.tsx 로직 분리
- 대상: `App.tsx`
- 내용:
  - 업로드/매칭/내보내기 핸들러를 커스텀 훅 또는 서비스로 분리
- 완료 기준:
  - 파일 길이 축소, 책임 분리 명확

### 8) 사용자 피드백 개선
- 대상: `App.tsx`
- 내용:
  - 에러 메시지 코드화
  - 처리 단계별 진행 상태 표시
- 완료 기준:
  - 실패 시 사용자가 다음 행동을 알 수 있음

### 9) 설정 상수화
- 대상: `App.tsx`, `services/*.ts`
- 내용:
  - 파일 크기 제한, 랜덤 범위, 청크 사이즈 등 설정 분리
- 완료 기준:
  - 매직넘버 제거, 조정 용이성 향상

---

## P3 (확장)

### 10) 매칭 규칙 고도화
- 대상: `services/excelService.ts`
- 내용:
  - 이름 정규화 규칙 설정화
  - 동명이인/이름 변형 대응 옵션
- 완료 기준:
  - 매칭 정확도 향상

### 11) 결과 리포트 내보내기
- 대상: 신규 서비스 파일
- 내용:
  - 매칭 성공/실패 목록 CSV 또는 JSON 출력
- 완료 기준:
  - 검수 가능한 결과 리포트 생성

### 12) 테스트 자동화 기본 세트
- 대상: 테스트 디렉터리 신규
- 내용:
  - `normalizeName`, `printArea 파서`, `placeholder 판별` 단위 테스트
- 완료 기준:
  - 핵심 유틸 회귀 테스트 통과
