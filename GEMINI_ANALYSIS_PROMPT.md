# Gemini 분석/보강 요청 프롬프트

아래 내용을 Gemini에 그대로 입력해서, 이 프로젝트를 분석하고 실제 보강 작업까지 이어서 진행해줘.

---

## 역할
너는 시니어 프론트엔드/문서 자동화 엔지니어다. 
대상은 React + TypeScript + ExcelJS 기반의 서명 자동 삽입 도구다.

## 목표
1. 현재 코드베이스를 진단한다.
2. 추가해야 할 기능과 보강 포인트를 우선순위로 제안한다.
3. 실제 코드 변경 계획(파일 단위)을 제시한다.
4. 각 항목에 대해 구현 난이도/효과/리스크를 같이 제시한다.

## 프로젝트 컨텍스트
- 스택: React, TypeScript, Vite, ExcelJS, html2canvas, jsPDF
- 핵심 기능:
  - XLSX 업로드/파싱
  - 서명 이미지 업로드
  - 이름 기반 자동 매칭
  - 엑셀/PDF/PNG 내보내기
- 주요 파일:
  - `App.tsx`: 전체 UI/상태/핸들러
  - `services/excelService.ts`: 파싱/자동매칭/최종 엑셀 생성
  - `services/alternativeExportService.ts`: PDF/PNG 내보내기
  - `services/excelUtils.ts`: 유틸 함수
  - `types.ts`: 타입 정의
  - `IMPROVEMENTS.md`, `TESTING_GUIDE.md`, `CHANGELOG.md`: 개선 및 테스트 문서

## 반드시 점검할 항목
아래 항목을 빠짐없이 진단해줘.

1) 아키텍처/응집도
- `App.tsx`에 로직이 과도하게 몰려있는지
- 컴포넌트 분리/서비스 경계가 적절한지

2) 성능/메모리
- 대용량 XLSX 처리 시 브라우저 메모리 사용량
- 이미지 회전/캔버스 처리 최적화 가능성
- Object URL 생성/해제 누락 가능성

3) 안정성/예외처리
- 파일 손상/형식 오류/엣지 케이스 대응 수준
- 병합셀/인쇄영역 보존 로직의 취약 지점
- 랜덤 매칭 로직의 재현성(테스트 가능성)

4) 타입 안정성/코드 품질
- `any` 사용 최소화 필요 지점
- 함수 분리, 중복 제거, 상수화 개선점

5) UX
- 사용자 에러 메시지의 명확성
- 처리 진행률/실패 원인 안내 개선
- 내보내기 옵션 사용 흐름 개선

6) 테스트 전략
- 반드시 필요한 단위 테스트
- 통합 테스트 시나리오
- 회귀 방지 핵심 테스트 목록

7) 보안/개인정보
- 로컬 처리 범위 명확화
- 콘솔 로그에 개인정보 노출 가능성

## 출력 형식 (반드시 준수)
다음 5개 섹션으로 결과를 작성해줘.

### A. 진단 요약 (10줄 이내)
- 현재 상태를 한눈에 요약

### B. 우선순위 개선안 Top 10
각 항목에 대해 아래 형식을 사용:
- 제목
- 문제
- 개선 방법
- 영향도(상/중/하)
- 구현 난이도(상/중/하)
- 예상 작업량(시간)

### C. 즉시 적용 Quick Wins (오늘 적용 가능한 5개)
- 작은 변경으로 효과 큰 항목만

### D. 파일 단위 구현 계획
예시 형식:
- `App.tsx`: 어떤 함수/상태를 어떻게 분리할지
- `services/excelService.ts`: 어떤 함수 추출/수정이 필요한지
- `types.ts`: 어떤 타입 보강이 필요한지

### E. 단계별 실행 로드맵
- 1단계(안정화)
- 2단계(성능)
- 3단계(UX/테스트)
각 단계별 완료 기준(DoD) 포함

## 추가 조건
- 과한 리라이팅 금지. 점진적 개선 우선.
- 기존 UX를 깨지 않는 범위에서 제안.
- 추상론 대신 파일/함수 기준의 구체적 액션으로 작성.
- 제안만 하지 말고, “바로 적용할 코드 변경 순서”까지 제시.
